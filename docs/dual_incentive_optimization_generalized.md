# Layer 2 전환 블록체인을 위한 동적 이중 인센티브 최적화 프레임워크

**저자**: Tokamak Network
**작성일**: 2025년 12월
**키워드**: 시뇨리지 분배, Layer 2 인센티브, 동적 전환 팩터, 이중 이해관계자 최적화, 블록체인 경제학

---

## 초록

블록체인 네트워크가 Layer 1에서 Layer 2 아키텍처로 전환할 때, 기존 스테이커와 신규 L2 운영자 간의 시뇨리지 보상 재분배라는 핵심 과제가 발생한다. 이 과정에서 이해관계자 이탈 없이 보상을 재분배하는 것이 중요하다. 본 논문은 L2 전환을 진행하는 모든 시뇨리지 발행 블록체인에 적용 가능한 일반화된 동적 시뇨리지 분배 프레임워크를 제시한다.

본 연구는 L2 생태계 성장 지표에 연동되어 자동으로 조정되는 동적 전환 팩터를 제안한다. 핵심 설계 원칙은 추가 시뇨리지(r)가 먼저 0이 된 후 지분 시뇨리지(λ)가 감소하는 것이다. 이를 통해 스테이커의 지분 인정을 최대한 보호하면서 점진적 전환을 달성한다. 제안 공식은 다음과 같다.

$$r(B) = r_0 \cdot \max\left(0, 1 - \frac{B}{B_r}\right)$$

$$\lambda(B) = \lambda_{min} + (1 - \lambda_{min}) \cdot \frac{k}{k + \max(0, B - B_r)}$$

여기서 B는 L2 생태계 성장 지표이다. 수학적 분석 결과, 제안 함수는 연속성, 단조성, 경계 조건 만족, Lyapunov 안정성을 보장한다. 선형 및 계단 함수 대비 시장 충격을 82-97% 감소시키며, 거버넌스 개입 없이 예측 가능한 전환을 제공한다. 실증 검증으로 토카막 네트워크 사례 연구를 수행하여 프레임워크의 실용성을 입증한다.

## 1. 서론

### 1.1 L1에서 L2로의 전환 과제

이더리움 생태계는 Layer 1에서 Layer 2 네트워크로의 전례 없는 경제 활동 이동을 목격하고 있다. 2025년 현재, Uniswap, Aave, Synthetix를 포함한 주요 프로토콜들이 L2 솔루션에 배포되었으며, 이더리움 L2의 총 예치 가치(TVL)는 2024년 말 500억 달러를 돌파하였고 2025년에는 300억~600억 달러 사이에서 변동하고 있다 [1].

이러한 전환은 근본적인 인센티브 재분배 문제를 야기한다. 네트워크를 보호해온 L1 스테이커들은 지속적인 보상을 기대하며, 인프라를 제공하는 L2 시퀀서와 검증자들은 자신들의 서비스에 대한 보상을 요구한다. 그러나 고정 발행량 네트워크에서 L2 보상은 기존 스테이커 할당분에서 확보해야 하는 제로섬 제약이 존재한다.

이러한 긴장을 해결하지 못하는 네트워크는 두 가지 실패 모드에 직면한다. 첫째, 공격적인 L2 보상 할당이 스테이커의 언스테이킹과 매도를 유발하는 스테이커 이탈이다. 둘째, 불충분한 L2 인센티브가 생태계 성장을 저해하는 운영자 부족이다.

### 1.2 현재 접근법과 한계

### 1.3 거버넌스 기반 조정의 한계

일부 네트워크는 거버넌스를 통한 수동 파라미터 조정으로 전환을 관리한다. 그러나 이 접근법은 여러 한계를 가진다. 정치적 불확실성으로 인해 스테이커와 L2 운영자 간의 이해충돌이 투표 과정에서 교착 상태를 야기할 수 있다. 또한 이산적 변경은 파라미터 변경 시점에 시장 충격을 야기하며, 투표 결과에 대한 예측 불가능성은 장기 계획을 어렵게 만든다.

### 1.4 연구 목표 및 적용 범위

본 연구는 거버넌스 기반 수동 조정을 L2 생태계 지표 연동 자동 조정으로 대체하는 메커니즘을 제안한다. 구체적으로, 지분 시뇨리지 팩터(λ)와 추가 시뇨리지율(r)이 L2 생태계 성장 지표에 연동되어 자동으로 조정되는 함수를 설계한다. 이를 통해 예측 가능하고 시장 친화적인 전환 경로를 제공하며, 스테이커와 L2 운영자 모두에게 합리적 인센티브를 보장한다.

본 프레임워크의 적용 범위는 **시뇨리지(인플레이션) 기반 보상 체계를 갖춘 네트워크**로 한정된다. 수수료 기반으로만 운영되는 L2나 고정 공급량 네트워크에는 직접 적용되지 않는다. 시뇨리지를 통해 스테이커에게 보상을 분배하고, 이 보상의 일부를 L2 운영자에게 재분배하는 구조를 가진 네트워크가 본 연구의 대상이다.

### 1.5 기여

본 논문의 주요 기여는 다음과 같다. 첫째, L2 생태계 지표에 연동된 자동 조정 메커니즘인 동적 전환 팩터 프레임워크를 제시한다. 둘째, 스테이커 지분 인정을 최대한 보호하는 r 선감소 원칙을 정립한다. 셋째, 연속성, 안정성, 시장 충격 감소에 대한 수학적 분석을 제공한다. 넷째, 선형, 계단, 쌍곡선 함수에 대한 정량적 비교 분석을 수행한다. 다섯째, 토카막 네트워크 사례 연구를 통한 실증 검증을 제시한다.

본 논문은 다음과 같이 구성된다. 2장에서 시스템 모델을 정의하고, 3장에서 동적 전환 팩터를 설계한다. 4장에서 후보 함수를 비교 분석하고, 5장에서 이해관계자별 영향을 분석한다. 6장에서 균형 조건을 도출하고, 7장에서 수학적 특성을 증명한다. 8장에서 토카막 네트워크 사례 연구를 수행하고, 9장에서 관련 연구를 검토한다. 10장에서 결론을 맺는다.

## 2. 시스템 모델

### 2.1 기본 시뇨리지 분배 구조

시뇨리지 발행 블록체인의 일반적인 분배 구조는 다음과 같이 모델링할 수 있다. 기간당 총 시뇨리지 발행량 $A_{total}$은 스테이커 분배분 $S_{staker}$, L2 운영자 분배분 $S_{L2}$, DAO 또는 재단 분배분 $S_{DAO}$로 구성된다.

$$A_{total} = S_{staker} + S_{L2} + S_{DAO}$$

이 구조에서 핵심 과제는 L2 생태계 성장에 따라 $S_{staker}$와 $S_{L2}$ 간의 비율을 어떻게 조정할 것인가이다.

### 2.2 이중 시뇨리지 모델

스테이커에게 분배되는 시뇨리지를 두 가지 구성요소로 분해하면 전환 메커니즘 설계가 용이해진다.

첫 번째 구성요소는 지분 시뇨리지(Staked Seigniorage)이다. 이는 스테이커가 예치한 지분에 비례하여 받는 기본 보상으로, 다음과 같이 정의된다.

$$S_{staked} = \lambda \cdot A_{total} \cdot \sigma$$

여기서 λ ∈ [0, 1]은 지분 시뇨리지 팩터이고, σ는 전체 공급량 대비 스테이킹 비율이다. λ = 1이면 스테이킹 지분에 비례하는 시뇨리지 전액이 스테이커에게 분배되며, λ가 감소하면 해당 비율만큼 다른 용도로 재분배된다.

두 번째 구성요소는 추가 시뇨리지(Relative Seigniorage)이다. 이는 지분 시뇨리지 이후 남은 재원에서 스테이커에게 추가로 분배되는 보너스 성격의 보상이다.

$$S_{relative} = (A_{total} - S_{staked}) \cdot r$$

여기서 r ∈ [0, 1]은 추가 시뇨리지율이다. 이 구성요소는 스테이커에게 추가적인 인센티브를 제공하지만, L2 전환 시 먼저 조정되어야 할 "보너스" 성격을 갖는다.

### 2.3 L2 생태계 지표

L2 생태계의 성장을 측정하기 위한 지표 B를 정의한다. B는 네트워크의 L2 생태계에 브릿지된 총 자산량으로, 각 L2의 브릿지된 자산량의 합으로 계산된다.

$$B = \sum_{i=1}^{n} B_i$$

B가 증가할수록 L2 생태계가 성장하고 있음을 의미하며, 이에 따라 전환 팩터가 조정되어야 한다. B를 전환 지표로 사용하는 근거는 다음과 같다. 첫째, B는 L2 생태계에 대한 실제 경제적 참여를 측정한다. 둘째, B는 L2 브릿지 컨트랙트에 실제 락업된 자산량이므로 조작이 구조적으로 어렵다. 셋째, B 증가는 L2 운영자에 대한 보상 필요성 증가와 자연스럽게 연결된다.

### 2.4 문제 정식화

본 연구의 목표는 B의 함수로서 λ(B)와 r(B)를 설계하여 다음 조건들을 동시에 만족하는 것이다.

경계 조건으로는 전환 전 상태에서 λ(0) = 1, r(0) = r_0이 성립해야 하며, 완전 전환 상태에서 λ(∞) = λ_min, r(∞) = 0이 성립해야 한다. 연속성 조건으로 λ와 r은 B에 대해 연속이어야 한다. 단조성 조건으로 λ와 r은 B에 대해 비증가해야 한다. 스테이커 보호 조건으로 r = 0이 될 때까지 λ = 1이 유지되어야 한다.

## 3. 동적 전환 팩터 설계

### 3.1 설계 원칙

동적 전환 팩터 설계의 첫 번째 원칙은 r 선감소, λ 후감소이다. 추가 시뇨리지(r)와 지분 시뇨리지(λ)는 경제적 성격이 근본적으로 다르다. 추가 시뇨리지는 스테이커에게 주어지는 "보너스" 성격의 보상으로, 전환 과정에서 먼저 감소시켜야 한다. 반면 지분 시뇨리지는 스테이커가 예치한 "원금 대비 보상"으로, 최대한 나중에 감소시켜야 한다.

이 원칙은 스테이커의 지분 인정을 최대한 보호한다. 스테이커가 예치한 자본에 대한 기본 보상(λ)은 유지하면서, 추가 보너스(r)를 먼저 조정하는 것이 경제적으로 공정하다. 이를 통해 스테이커는 L2 생태계 초기 성장 단계에서도 자신의 지분에 대한 온전한 인정을 받을 수 있다.

두 번째 원칙은 연속성과 예측 가능성이다. 불연속적인 파라미터 변화는 시장 충격을 야기한다. 계단 함수 형태의 거버넌스 기반 조정은 변경 시점에 급격한 가격 변동을 초래할 수 있다. 전환 팩터는 B에 대해 연속이어야 하며, 참여자들이 B 값을 관찰함으로써 미래 상태를 예측할 수 있어야 한다.

세 번째 원칙은 경계 조건 만족이다. B = 0일 때 전환 전 상태가 유지되어야 하며(λ = 1, r = r_0), B → ∞일 때 완전 전환 상태에 도달해야 한다(λ = λ_min, r = 0).

### 3.2 제안 공식

위의 설계 원칙을 만족하는 동적 전환 팩터 공식을 제안한다.

추가 시뇨리지율 r(B)는 다음과 같이 정의된다.

$$r(B) = r_0 \cdot \max\left(0, 1 - \frac{B}{B_r}\right)$$

이 공식에서 $r_0$는 초기 추가 시뇨리지율이고, $B_r$은 r이 0이 되는 임계점이다. B < B_r 구간에서 r은 선형으로 감소하며, B ≥ B_r이 되면 r = 0이 되어 추가 시뇨리지가 완전히 제거된다.

지분 시뇨리지율 λ(B)는 다음과 같이 정의된다.

$$\lambda(B) = \lambda_{min} + (1 - \lambda_{min}) \cdot \frac{k}{k + \max(0, B - B_r)}$$

이 공식에서 $\lambda_{min}$은 최소 지분 시뇨리지율(하한)이고, $k$는 반포화 상수이다. B < B_r 구간에서 max(0, B - B_r) = 0이므로 λ = 1이 유지된다. 이는 r이 감소하는 동안 스테이커의 지분 시뇨리지를 완전히 보호한다. B = B_r 시점에서 r = 0이 되고 λ = 1인 상태에서 비로소 λ의 감소가 시작된다. B > B_r 구간에서 λ는 쌍곡선적으로 감소하며, B → ∞일 때 λ → λ_min으로 점근 수렴한다.

### 3.3 파라미터 해석

제안 공식의 네 가지 파라미터는 각각 고유한 역할을 수행한다.

$r_0$는 초기 추가 시뇨리지율로, 전환 전 스테이커가 받는 추가 보상 수준을 결정한다. 일반적으로 현재 시스템의 추가 시뇨리지율을 그대로 사용한다.

$B_r$은 r 제거 임계점으로, 추가 시뇨리지가 완전히 제거되는 시점이자 지분 시뇨리지 감소가 시작되는 시점이다. 이 값은 L2 생태계의 초기 성장 목표에 맞춰 설정한다.

$\lambda_{min}$은 최소 지분 시뇨리지 팩터로, 스테이커 APY의 하한을 결정한다. 목표 최소 APY에서 다음과 같이 역산할 수 있다.

$$\lambda_{min} = \frac{APY_{min} \cdot S}{A_{total} \cdot \sigma}$$

$k$는 반포화 상수로, λ의 감소 속도를 조절한다. B = B_r + k일 때 λ가 1과 λ_min의 중간값에 도달하므로, k는 "스테이킹 경제 규모 대비 L2 경제 규모가 어느 정도 비율이 되면 전환이 절반 진행되는가"를 결정한다. k 설정은 목표 전환 기간, 스테이커 이탈 임계점, L2 운영자 최소 보상 등 여러 요소를 고려하여 결정해야 하며, 구체적인 방법론은 3.5절에서 다룬다.

### 3.4 수학적 검증

제안 공식이 설계 원칙을 만족하는지 수학적으로 검증한다.

경계 조건 검증을 위해 B = 0일 때를 살펴보면, r(0) = r_0 × max(0, 1 - 0) = r_0이고, λ(0) = λ_min + (1 - λ_min) × k/(k + 0) = 1이 성립한다.

B = B_r일 때, r(B_r) = r_0 × max(0, 0) = 0이고, λ(B_r) = λ_min + (1 - λ_min) × k/(k + 0) = 1이 성립한다.

B → ∞일 때, r(∞) = 0이고, λ(∞) = λ_min + 0 = λ_min이 성립한다.

r 선감소 원칙의 검증을 위해 다양한 B 값에서 r(B)와 λ(B)의 값을 확인한다. B = 0에서 r = r_0, λ = 1로 r만 감소 중이고 λ는 유지된다. B = 0.5B_r에서 r = 0.5r_0, λ = 1로 여전히 r만 감소 중이다. B = B_r에서 r = 0, λ = 1로 r 감소가 완료되고 λ 감소가 시작되는 시점이다. B = 2B_r에서 r = 0, λ < 1로 이제 λ가 감소 중이다. B → ∞에서 r = 0, λ = λ_min으로 최종 상태에 도달한다.

이로써 r이 완전히 0이 된 후에야 λ가 감소를 시작한다는 r 선감소 원칙이 만족됨을 확인할 수 있다.

### 3.5 파라미터 결정 방법론

제안 프레임워크의 네 가지 파라미터(r_0, B_r, λ_min, k)는 네트워크의 경제적 목표와 제약 조건에 따라 체계적으로 결정되어야 한다. 본 절에서는 각 파라미터의 결정 방법론과 근거를 제시한다.

#### 3.5.1 r_0 (초기 추가 시뇨리지율)

r_0는 전환 전 시스템의 현재 추가 시뇨리지율을 그대로 사용한다. 이는 전환 시작 시점에서 스테이커 보상의 연속성을 보장하기 위함이다. 기존 시스템에서 r_0 = 0.4라면, 제안 프레임워크에서도 B = 0일 때 동일한 보상 수준을 유지한다.

#### 3.5.2 λ_min (최소 지분 시뇨리지율)

λ_min은 완전 전환 상태에서 스테이커에게 보장하는 최소 APY로부터 역산한다. 목표 최소 APY를 $APY_{min}$이라 하면 다음과 같이 계산된다.

$$\lambda_{min} = \frac{APY_{min} \cdot S}{A_{total} \cdot \sigma}$$

λ_min 설정 시 고려해야 할 요소는 다음과 같다. 첫째, 경쟁 프로토콜의 스테이킹 APY이다. ETH 2.0이 4-5%, Solana가 6-8%, Cosmos가 15-20% 수준이므로, 완전 전환 후에도 이들과 경쟁력 있는 APY를 제공해야 스테이커 이탈을 방지할 수 있다. 둘째, 토큰 경제 안정성 유지를 위한 최소 스테이킹 비율이다. APY가 너무 낮아지면 스테이커가 이탈하여 스테이킹 비율이 감소하고, 이는 유통량 증가로 인한 매도 압력을 야기할 수 있다.

#### 3.5.3 B_r (r 제거 임계점)

B_r은 추가 시뇨리지(r)가 완전히 제거되고 지분 시뇨리지(λ) 감소가 시작되는 임계점이다. 이 값은 L2 생태계의 초기 성장 목표와 스테이커 보호 기간을 고려하여 설정한다.

B_r 설정 시 고려해야 할 요소는 다음과 같다. 첫째, L2 생태계 초기 성장 목표이다. B_r은 "L2 생태계가 이 정도 규모에 도달하면 본격적인 전환을 시작한다"는 의미를 갖는다. 네트워크의 L2 로드맵과 성장 전략에 따라 설정한다. 둘째, 스테이커 보호 기간이다. B < B_r 구간에서는 λ = 1이 유지되어 스테이커의 지분 인정이 100% 보장된다. B_r이 클수록 이 보호 기간이 길어진다. 셋째, 예상 L2 성장률이다. 연간 L2 브릿지 증가량을 $\Delta B_{year}$라 하면, B_r에 도달하는 데 걸리는 시간은 $T_r = B_r / \Delta B_{year}$이다. 이 기간이 너무 짧으면 스테이커에게 충격이 되고, 너무 길면 L2 재원 확보가 지연된다.

#### 3.5.4 k (반포화 상수)

k는 λ의 감소 속도를 결정하는 핵심 파라미터로, "스테이킹 경제 규모 대비 L2 경제 규모가 어느 정도 비율이 되면 전환이 절반 진행되는가"를 의미한다. B = B_r + k일 때 λ는 1과 λ_min의 중간값에 도달한다.

$$\lambda(B_r + k) = \lambda_{min} + (1 - \lambda_{min}) \cdot \frac{k}{2k} = \frac{1 + \lambda_{min}}{2}$$

k 설정을 위한 세 가지 접근법을 제시한다.

**접근법 1: 스테이커 이탈 임계점 기반**

스테이커가 이탈하기 시작하는 APY 임계점을 $APY_{threshold}$라 하자. 경쟁 프로토콜 대비 매력적인 APY를 유지해야 스테이커 이탈을 방지할 수 있다. 이 APY에 해당하는 λ 값을 $\lambda_{threshold}$라 하면 다음과 같이 계산된다.

$$\lambda_{threshold} = \frac{APY_{threshold} \cdot S}{A_{total} \cdot \sigma}$$

λ(B) = λ_threshold가 되는 B 값을 $B_{threshold}$라 하면, λ(B) 공식에서 역산할 수 있다.

$$B_{threshold} = B_r + k \cdot \frac{1 - \lambda_{threshold}}{\lambda_{threshold} - \lambda_{min}}$$

이 접근법의 핵심은 "L2 브릿지량이 $B_{threshold}$에 도달했을 때 스테이커 APY가 이탈 임계점에 도달한다"는 관계를 설정하는 것이다. 네트워크는 $B_{threshold}$가 충분히 크도록 k를 설정하여, L2 생태계가 충분히 성장한 후에야 스테이커 이탈 압력이 발생하도록 조절할 수 있다.

**접근법 2: L2 운영자 최소 보상 기반**

L2 운영자가 참여하기 위한 최소 연간 보상을 $R_{min}$이라 하자. L2 분배 재원 $A_{L2}(B)$가 $R_{min}$을 초과하는 B 값을 계산하고, 이 시점에 적절한 전환 진행률이 달성되도록 k를 설정한다.

**접근법 3: 스테이킹량 비율 기반 (경험적 접근)**

k를 현재 스테이킹량 S의 비율로 설정하는 경험적 접근법이다. 이 방법은 네트워크 규모에 자동으로 비례하는 장점이 있다.

$$k = \alpha \cdot S, \quad \alpha \in [0.3, 0.5]$$

α = 0.3 (빠른 전환): L2 브릿지량이 스테이킹량의 30%에 도달하면 전환 절반 진행. L2 성장을 촉진하고자 할 때 적합하다.

α = 0.5 (보수적 전환): L2 브릿지량이 스테이킹량의 50%에 도달해야 전환 절반 진행. 스테이커 보호를 강화하고자 할 때 적합하다.

이 범위의 근거는 다음과 같다. α < 0.3이면 전환이 너무 빨라 스테이커 충격이 발생할 수 있다. α > 0.5이면 전환이 너무 느려 L2 재원 확보가 지연된다. 0.3-0.5 범위는 스테이커 보호와 L2 성장 인센티브 사이의 균형점이다.

#### 3.5.5 파라미터 간 상호작용

세 파라미터(B_r, λ_min, k)는 독립적이지 않으며 상호 영향을 미친다. B_r과 k의 합 B_r + k는 λ 감소의 절반이 완료되는 총 브릿지량을 결정한다. λ_min과 k는 함께 스테이커 APY의 감소 경로를 결정한다. B_r은 r이 0에 도달하는 시점을 결정하며, B_r이 작으면 r이 빠르게 제거되고 λ 감소가 일찍 시작된다.

따라서 파라미터 설정 시 개별 파라미터만 고려하지 않고, 전체 전환 경로를 시뮬레이션하여 스테이커 APY 곡선과 L2 재원 곡선이 목표에 부합하는지 확인해야 한다.

#### 3.5.6 민감도 분석의 필요성

실제 네트워크에 적용하기 전에 각 파라미터에 대한 민감도 분석을 수행해야 한다. 특히 k 값에 대해서는 k = 0.2S, 0.3S, 0.4S, 0.5S 등 여러 시나리오에서 스테이커 APY 변화, L2 재원 변화, 예상 전환 기간을 비교 분석해야 한다. 이를 통해 네트워크의 목표와 제약 조건에 가장 부합하는 파라미터 조합을 선택할 수 있다.

## 4. 전환 함수 비교 분석

### 4.1 후보 함수

제안 함수의 적합성을 평가하기 위해 선형 함수와 계단 함수를 비교 대상으로 설정한다.

선형 함수는 B에 비례하여 일정한 속도로 감소하는 형태이다.

$$\lambda_L(B) = \max\left(\lambda_{min}, 1 - \frac{(1-\lambda_{min}) \cdot B}{B_{max}}\right)$$

이 함수는 단순하고 직관적이지만, B가 작을 때부터 즉시 λ가 감소하기 시작하여 r 선감소 원칙을 만족하지 못한다.

계단 함수는 거버넌스 결정에 따른 이산적 조정을 나타낸다.

$$\lambda_S(B) = \begin{cases} 1 & B < B_1 \\ \lambda_1 & B_1 \leq B < B_2 \\ \lambda_{min} & B \geq B_2 \end{cases}$$

이 함수는 특정 임계점에서 급격한 변화가 발생하여 시장 충격을 야기한다.

제안 함수는 쌍곡선 형태로, r = 0 이후 점진적으로 감소한다.

$$\lambda_H(B) = \lambda_{min} + (1 - \lambda_{min}) \cdot \frac{k}{k + \max(0, B - B_r)}$$

### 4.2 요구사항 충족도 비교

다섯 가지 요구사항에 대한 각 함수의 충족 여부를 분석한다.

초기 호환성(λ(0) = 1) 요구사항은 세 함수 모두 만족한다.

점근적 하한 요구사항은 λ가 B → ∞일 때 λ_min으로 수렴하는지를 의미한다. 선형 함수는 B_max에서 정확히 λ_min에 도달한 후 일정하므로 점근적 수렴이 아니다. 계단 함수도 특정 임계점 이후 일정값으로 고정된다. 제안 함수만이 λ_min으로 점근 수렴한다.

연속성 요구사항은 선형 함수와 제안 함수가 만족하지만, 계단 함수는 불연속점이 존재한다.

r 선감소 보장 요구사항은 제안 함수만이 만족한다. 선형 함수와 계단 함수는 λ와 r의 감소 순서를 제어하지 않는다.

자동 조정 요구사항은 B에 연동된 자동 조정 여부를 의미한다. 선형 함수와 제안 함수는 만족하지만, 계단 함수는 거버넌스 투표에 의존한다.

종합하면, 선형 함수는 5개 중 3개, 계단 함수는 5개 중 1개, 제안 함수는 5개 중 5개 요구사항을 만족한다.

### 4.3 시장 충격 분석

시장 충격은 단일 기간 내 최대 보상 변화율로 정의한다.

$$\text{Market Shock} = \max_B \left| \frac{d(APY)}{dB} \cdot \Delta B \right|$$

기간당 예상 브릿지량 δ를 가정하고 각 함수의 시장 충격을 비교한다.

계단 함수는 임계점에서 10-30%의 급격한 충격이 발생하며, 평균 충격은 약 10%이다. 충격 패턴은 이산적 점프 형태이다.

선형 함수는 전 구간에서 일정한 충격이 발생하며, 그 크기는 중간 수준이다. 충격 패턴은 균일하다.

제안 함수는 초기에 낮은 충격으로 시작하여 B 증가에 따라 충격이 점진적으로 감소한다. 이는 제안 함수의 λ 미분이 분모에 $(k + B - B_r)^2$를 포함하여 B 증가에 따라 크기가 감소하기 때문이다.

정량적으로, 제안 함수의 시장 충격은 선형 함수 대비 약 82%, 계단 함수 대비 약 97% 감소한다.

### 4.4 함수 선택 근거

제안 함수가 최적의 선택인 이유를 정리하면 다음과 같다.

첫째, 초기 안정성 측면에서 B < B_r 구간에서 λ = 1을 유지함으로써 스테이커를 보호한다.

둘째, 점진적 전환 측면에서 불연속 점프 없이 연속적인 전환을 제공한다.

셋째, 자연스러운 수렴 측면에서 λ_min으로의 점근 수렴은 완전 전환 상태에 도달하지만 그 과정이 무한히 점진적임을 보장한다.

넷째, 시장 친화성 측면에서 B 증가에 따라 충격이 감소하여, L2 생태계가 성숙할수록 더 안정적인 전환이 이루어진다.

## 5. 이해관계자 분석

### 5.1 스테이커 분석

스테이커의 연간 수익률(APY)은 지분 시뇨리지와 추가 시뇨리지의 합으로 계산된다.

$$APY_S(B) = \frac{A_{total} \cdot \left[\lambda(B) \cdot \sigma + r(B) \cdot (1 - \lambda(B) \cdot \sigma)\right]}{S}$$

여기서 S는 총 스테이킹량이다.

스테이커 APY의 변화 패턴은 두 단계로 나뉜다. 첫 번째 단계(B < B_r)에서는 r만 감소하고 λ = 1이 유지된다. 이 구간에서 APY 감소폭은 제한적이며, 스테이커는 자신의 지분에 대한 온전한 인정을 받는다. 두 번째 단계(B ≥ B_r)에서는 r = 0이 되고 λ가 감소하기 시작한다. APY는 점진적으로 감소하여 최종적으로 APY_min에 수렴한다.

특히 r = 0이 되는 시점(B = B_r)에서의 스테이커 APY를 계산하면 다음과 같다.

$$APY_S(B_r) = \frac{A_{total} \cdot \lambda(B_r) \cdot \sigma}{S} = \frac{A_{total} \cdot \sigma}{S}$$

이는 추가 시뇨리지 없이 지분 시뇨리지만으로 받는 APY로, 스테이커 지분 인정이 100% 유지됨을 의미한다.

### 5.2 L2 운영자 분석

L2 운영자에게 분배되는 재원은 스테이커 시뇨리지를 제외한 나머지에서 DAO 분배를 뺀 금액이다.

$$A_{L2}(B) = A_{total} \cdot (1 - \lambda(B) \cdot \sigma) \cdot (1 - r(B)) \cdot (1 - d)$$

여기서 d는 DAO 분배 비율이다.

L2 분배 재원의 변화 패턴도 두 단계로 나뉜다. 첫 번째 단계(B < B_r)에서는 r이 감소함에 따라 $(1 - r(B))$ 항이 증가하여 A_L2가 증가한다. 두 번째 단계(B ≥ B_r)에서는 λ가 감소함에 따라 $(1 - \lambda(B) \cdot \sigma)$ 항이 증가하여 A_L2가 추가로 증가한다.

중요한 특성으로, L2 분배 재원은 B에 대해 단조 증가한다. 이는 L2 생태계 성장이 L2 운영자 보상 증가로 직접 연결됨을 의미하며, 운영자에게 L2 생태계 성장에 기여할 강력한 인센티브를 제공한다.

### 5.3 경쟁력 분석

제안 프레임워크를 적용한 네트워크의 스테이커 APY가 시장에서 경쟁력을 갖는지 분석한다. 주요 프로토콜의 스테이킹 APY를 살펴보면, ETH 2.0은 4-5%, Solana는 6-8%, Cosmos는 15-20%, Polkadot은 12-15% 수준이다.

제안 프레임워크는 초기(B = 0)에 높은 APY를 제공하다가 L2 생태계 성장에 따라 점진적으로 감소한다. λ_min을 적절히 설정하면, 완전 전환 상태에서도 ETH 2.0 수준의 경쟁력 있는 APY를 유지할 수 있다. 중간 전환 구간에서는 Cosmos나 Polkadot 수준의 APY가 유지되어 스테이커 이탈을 방지한다.

## 6. 균형 분석

### 6.1 파레토 효율성

파레토 균형은 한 이해관계자의 효용을 감소시키지 않고는 다른 이해관계자의 효용을 증가시킬 수 없는 상태를 의미한다.

제안 프레임워크에서 B는 시장 참여자들의 자유로운 선택에 의해 결정되며, 각 B 값에서의 분배는 공식에 의해 자동으로 결정된다. 총 시뇨리지 $A_{total}$이 고정된 상태에서 스테이커와 L2 운영자 분배의 합은 항상 일정하다(DAO 분배 제외). 따라서 한쪽의 분배 증가는 필연적으로 다른 쪽의 감소를 의미하며, 각 B 값에서의 분배는 파레토 효율적이다.

### 6.2 균형 구간

양측 이해관계자가 모두 합리적 인센티브를 갖는 B의 범위를 균형 구간이라 정의한다. 균형 구간이 성립하려면 두 가지 조건이 충족되어야 한다.

첫째, 스테이커 APY가 대체 투자 수익률보다 높아야 한다. 거래소 및 DeFi 플랫폼에서 제공하는 수익률과 비교하여 매력적인 APY를 유지해야 스테이커 이탈을 방지할 수 있다.

둘째, L2 운영자 보상이 운영 비용보다 높아야 한다. 시퀀서 운영에 필요한 인프라 비용을 충당하고 이익을 남길 수 있어야 운영자 참여가 유지된다.

이 두 조건을 만족하는 B의 범위가 균형 구간이며, 이 구간에서 네트워크는 스테이커 이탈과 운영자 부족 모두를 피할 수 있다.

### 6.3 내쉬 균형 분석

스테이커와 L2 운영자의 전략적 상호작용을 게임 이론적으로 분석한다.

스테이커는 스테이킹 유지와 언스테이킹 중에서 선택한다. 스테이킹 유지의 보수는 APY_S이고, 언스테이킹의 보수는 기회비용(대안 투자 수익률)이다.

L2 운영자는 L2 운영과 미운영 중에서 선택한다. L2 운영의 보수는 L2 보상에서 운영 비용을 뺀 값이고, 미운영의 보수는 0이다.

균형 구간에서 APY_S > 기회비용이고 L2 보상 > 운영비용이 성립한다면, (스테이킹 유지, L2 운영)은 내쉬 균형이 된다. 이 상태에서 어느 한 참여자도 일방적으로 전략을 변경하여 더 높은 보수를 얻을 수 없다.

## 7. 수학적 특성

### 7.1 연속성

제안 함수 λ(B)와 r(B)가 B ∈ [0, ∞)에서 연속임을 증명한다.

r(B)의 경우, B < B_r에서는 선형 함수 $r_0(1 - B/B_r)$이고, B ≥ B_r에서는 상수 0이다. B = B_r에서의 연속성을 확인하면, 좌극한 $\lim_{B \to B_r^-} r(B) = r_0 \cdot 0 = 0$이고 함수값 $r(B_r) = 0$이므로 연속이다.

λ(B)의 경우, B < B_r에서는 상수 1이고, B ≥ B_r에서는 쌍곡선 함수이다. B = B_r에서의 연속성을 확인하면, 좌극한 $\lim_{B \to B_r^-} \lambda(B) = 1$, 우극한 $\lim_{B \to B_r^+} \lambda(B) = \lambda_{min} + (1-\lambda_{min}) \cdot 1 = 1$, 함수값 $\lambda(B_r) = 1$이므로 연속이다.

### 7.2 미분가능성

λ(B)는 B = B_r에서 미분가능하지 않다. 좌미분은 $\frac{d\lambda}{dB}|_{B_r^-} = 0$이고, 우미분은 $\frac{d\lambda}{dB}|_{B_r^+} = -\frac{(1-\lambda_{min})}{k} < 0$이다.

그러나 이 미분불가능점은 연속성을 해치지 않으며, 실용적으로 문제가 되지 않는다. 오히려 이 지점이 r = 0과 λ 감소 시작이 동시에 일어나는 전환점으로서 의미를 갖는다.

### 7.3 Lyapunov 안정성

에너지 함수 $V(\lambda) = (\lambda - \lambda_{min})^2$를 정의하면, λ_min이 Lyapunov 의미에서 점근적으로 안정함을 증명할 수 있다.

B > B_r에서 에너지 함수의 B에 대한 미분은 다음과 같다.

$$\frac{dV}{dB} = 2(\lambda - \lambda_{min}) \cdot \frac{d\lambda}{dB}$$

λ > λ_min이고 $\frac{d\lambda}{dB} < 0$이므로, $\frac{dV}{dB} < 0$이 성립한다. 이는 에너지 함수가 B 증가에 따라 단조 감소함을 의미하며, λ는 λ_min으로 점근 수렴한다.

### 7.4 수렴 속도

λ(B)의 λ_min으로의 수렴은 $O(1/B)$ 속도를 갖는다.

B >> B_r + k일 때, λ(B) - λ_min의 점근적 행동을 분석하면 다음과 같다.

$$\lambda(B) - \lambda_{min} = (1-\lambda_{min}) \cdot \frac{k}{k + B - B_r} \sim \frac{(1-\lambda_{min}) \cdot k}{B}$$

따라서 수렴 속도는 $O(1/B)$이다. 이는 비교적 빠른 수렴을 의미하지만, B가 무한대로 갈 때까지 완전 수렴하지 않으므로 항상 λ > λ_min을 유지한다.

## 8. 실증 검증: 토카막 네트워크 사례 연구

### 8.1 네트워크 개요

토카막 네트워크는 TON 토큰을 발행하는 시뇨리지 기반 블록체인으로, Titan L2를 중심으로 L2 생태계를 구축 중이다. 2025년 현재 네트워크의 주요 지표는 다음과 같다. 연간 시뇨리지 발행량은 10,308,816 TON이며, 총 스테이킹량은 27,234,159 TON이다. 스테이킹 비율(σ)은 27.4%이고, 현재 스테이커 APY는 약 21%이다.

토카막 네트워크는 현재 스테이커에게만 시뇨리지를 분배하는 V2 모델을 운영하고 있다. V3 백서에서는 L2 운영자에게만 시뇨리지를 분배하는 방식을 제안하고 있으나, 이는 스테이커와 L2 운영자 간의 급격한 전환을 초래할 수 있다. 본 사례 연구에서는 제안 프레임워크를 토카막 네트워크에 적용하여 점진적 전환을 달성했을 때의 효과를 시뮬레이션한다.

### 8.2 파라미터 설정

제안 프레임워크를 토카막 네트워크에 적용하기 위한 파라미터를 다음과 같이 설정한다.

초기 추가 시뇨리지율 r_0는 현재 V2 시스템의 값인 0.4로 설정한다. r 제거 임계점 B_r은 L2 생태계 초기 성장 목표인 2천만 TON으로 설정한다. 최소 지분 시뇨리지율 λ_min은 최소 APY 5%를 보장하기 위해 0.5로 설정한다. 반포화 상수 k는 현재 스테이킹량의 약 37%인 천만 TON으로 설정한다.

λ_min = 0.5의 근거를 계산하면 다음과 같다. 목표 최소 APY를 5%로 설정하고, 공식 $\lambda_{min} = \frac{APY_{min} \cdot S}{A_{total} \cdot \sigma}$에 대입하면 $\lambda_{min} = \frac{0.05 \times 27,234,159}{10,308,816 \times 0.274} \approx 0.48$이 된다. 실용적 편의를 위해 0.5로 설정한다.

### 8.3 시뮬레이션 결과

다양한 B 값에서의 시뮬레이션 결과를 분석한다.

B = 0일 때, r(B) = 0.40, λ(B) = 1.00이며 스테이커 APY는 21.0%이다. L2 분배 재원은 0이다. 이는 전환 전 상태로, 현재 V2와 동일하다.

B = 천만 TON일 때, r(B) = 0.20, λ(B) = 1.00이며 스테이커 APY는 17.6%이다. L2 분배 재원은 358만 TON이다. r이 절반으로 감소했지만 λ는 여전히 1을 유지한다.

B = 2천만 TON(= B_r)일 때, r(B) = 0, λ(B) = 1.00이며 스테이커 APY는 14.1%이다. L2 분배 재원은 594만 TON이다. 이 시점에서 r이 완전히 0이 되고, λ의 감소가 시작된다.

B = 3천만 TON일 때, r(B) = 0, λ(B) = 0.75이며 스테이커 APY는 10.6%이다. L2 분배 재원은 677만 TON이다. λ가 감소하기 시작하여 L2 재원이 더 증가했다.

B = 5천만 TON일 때, r(B) = 0, λ(B) = 0.625이며 스테이커 APY는 8.8%이다. L2 분배 재원은 740만 TON이다.

B = 1억 TON일 때, r(B) = 0, λ(B) = 0.56이며 스테이커 APY는 7.9%이다. L2 분배 재원은 782만 TON이다.

시뮬레이션 결과에서 몇 가지 중요한 관찰을 도출할 수 있다. 첫째, r 선감소 원칙이 확인된다. B < 2천만 TON 구간에서 λ = 1이 유지된다. 둘째, 점진적 전환이 이루어진다. 스테이커 APY가 21%에서 시작하여 최소 5%까지 점진적으로 감소한다. 셋째, L2 재원이 단조 증가한다. B 증가에 따라 L2 분배 재원이 0에서 약 780만 TON까지 증가한다. 넷째, 균형 구간이 존재한다. B = 2천만~5천만 TON 구간에서 스테이커 APY 8.8%~14.1%, L2 재원 594만~740만 TON으로 양측 모두 합리적 인센티브가 확보된다.

### 8.4 기존 모델들과의 비교

토카막 네트워크의 현재 V2 모델, V3 백서 제안, 그리고 본 논문의 제안 프레임워크를 비교 분석한다.

현재 운영 중인 V2 모델에서는 시뇨리지가 전적으로 스테이커에게 분배된다. λ와 r 개념이 명시적으로 분리되어 있지 않으며, 모든 시뇨리지 보상이 스테이킹 참여자에게 돌아간다. 이 모델은 L2 운영자에 대한 인센티브가 없어 L2 생태계 성장에 한계가 있다.

V3 백서에서는 시뇨리지를 L2 운영자에게만 분배하는 방식을 제안한다. 이는 L2 생태계 성장에는 유리하지만, 기존 스테이커의 보상이 즉시 중단되어 급격한 이탈을 초래할 수 있다.

본 논문의 제안 프레임워크는 λ(B)와 r(B) 함수를 통해 L2 생태계 성장에 연동하여 자동으로 전환되는 메커니즘을 제공한다. 스테이커 보상은 점진적으로 감소하고 L2 운영자 보상은 점진적으로 증가하여, 양측 이해관계자 모두에게 예측 가능한 전환 경로를 제공한다.

y(x) L2 분배 함수, DAO 분배 공식, 검증자 분배 비율 등 다른 구성요소는 V3 백서의 설계를 그대로 활용할 수 있다. 본 논문의 제안 프레임워크는 V2와 V3 사이의 합리적인 해결점을 제시하며, 스테이커와 L2 운영자 모두에게 설득력 있는 전환 방안을 도출할 수 있다.

### 8.5 구현 예시

제안 프레임워크를 스마트 컨트랙트로 구현하는 예시는 다음과 같다.

```solidity
// 기존: 거버넌스가 직접 설정
uint256 public stakedSeigFactor;   // λ
uint256 public relativeSeigRate;   // r

// 제안: B에서 자동 계산
function getStakedSeigFactor() public view returns (uint256) {
    uint256 B = getTotalBridgedTON();
    if (B < B_r) {
        return RAY; // λ = 1
    }
    // λ(B) = λ_min + (1 - λ_min) · k/(k + B - B_r)
    uint256 excess = B - B_r;
    return lambda_min + rmul(RAY - lambda_min, rdiv(k, k + excess));
}

function getRelativeSeigRate() public view returns (uint256) {
    uint256 B = getTotalBridgedTON();
    if (B >= B_r) {
        return 0;
    }
    // r(B) = r_0 · (1 - B/B_r)
    return rmul(r_0, RAY - rdiv(B, B_r));
}
```

이 구현에서 기존의 고정 파라미터 stakedSeigFactor와 relativeSeigRate 대신 getTotalBridgedTON()에서 B 값을 읽어와 실시간으로 계산하는 함수로 대체한다.

## 9. 관련 연구

### 9.1 토큰 이코노믹스

Buterin 등은 ETH 2.0의 스테이킹 보상을 스테이킹 비율에 기반하여 설계하였다 [2]. 이 연구는 스테이킹 참여율에 따른 보상 조정 메커니즘을 제시했으나, L2 운영자에 대한 보상은 다루지 않았다. Roughgarden은 트랜잭션 수수료 메커니즘 설계를 게임 이론적으로 분석하였으나 [3], 시뇨리지 분배는 연구 범위 밖이었다.

### 9.2 동적 조정 메커니즘

MakerDAO의 안정화 수수료는 DAI 가격에 연동되어 자동으로 조정되는 선례를 제공한다 [4]. Compound의 이자율 곡선은 활용도에 따라 대출 이자율이 자동 조정되며 [5], Aave도 유사한 동적 이자율 모델을 채택한다 [6]. 본 연구의 λ(B), r(B) 공식은 이러한 자동 조정 메커니즘을 시뇨리지 분배에 최초로 적용한 것이다.

### 9.3 L2 경제학

Optimism과 Arbitrum은 각각의 L2 경제 모델을 제시하였으나 [7,8], 시퀀서 보상은 중앙화된 팀에 귀속되는 구조이다. Plasma Group은 탈중앙화된 L2 구조를 제안하였으나 [9], 시뇨리지 연동 인센티브 메커니즘은 다루지 않았다.

### 9.4 본 연구의 차별점

기존 연구와 비교하여 본 연구의 차별점을 정리하면 다음과 같다.

ETH 2.0 연구는 L2 보상을 고려하지 않았으나, 본 연구는 스테이커와 L2 운영자의 이중 인센티브를 동시에 설계한다. Optimism/Arbitrum 모델은 중앙화 시퀀서에 의존하나, 본 연구는 탈중앙화된 운영자 참여를 가능하게 한다. 거버넌스 기반 전환은 정치적 불확실성을 수반하나, 본 연구는 B 연동 자동 전환으로 이를 제거한다. DeFi 이자율 모델은 대출 시장에 특화되어 있으나, 본 연구는 이러한 자동 조정 개념을 시뇨리지 분배에 적용한다.

## 10. 결론

### 10.1 요약

본 논문은 L2 전환 블록체인을 위한 동적 이중 인센티브 최적화 프레임워크를 제안하였다.

핵심 기여로서, 첫째 L2 생태계 지표 B에 연동된 λ(B), r(B) 공식을 설계하여 거버넌스 개입 없는 자동 전환을 가능하게 하였다. 둘째, r 선감소 원칙을 정립하여 스테이커의 지분 인정을 최대한 보호하면서 점진적 전환을 달성하였다. 셋째, 연속성, 단조성, Lyapunov 안정성에 대한 수학적 분석을 통해 프레임워크의 이론적 견고성을 입증하였다. 넷째, 선형 및 계단 함수 대비 82-97%의 시장 충격 감소를 정량적으로 입증하였다. 다섯째, 토카막 네트워크 사례 연구를 통해 프레임워크의 실용적 적용 가능성을 검증하였다.

### 10.2 한계 및 향후 연구

본 연구의 한계로는 시뇨리지 기반 보상 체계를 갖춘 네트워크에만 적용 가능하다는 점, 단일 생태계 지표(B)에 의존한다는 점, 외부 시장 조건을 명시적으로 반영하지 않는다는 점이 있다. 수수료 기반 L2나 고정 공급량 네트워크에 대한 확장은 별도의 연구가 필요하다.

향후 연구 방향으로는 TVL, 거래량 등 다중 지표 연동 확장, 시장 조건에 따른 동적 k 조정 메커니즘, 게임 이론 기반 조작 저항성 분석, 다중 L2 환경에서의 분배 확장 등이 있다.

### 10.3 결론

제안 프레임워크는 거버넌스 기반 수동 조정의 불확실성을 제거하고, 예측 가능하며 시장 친화적인 L2 전환 경로를 제공한다. r 선감소 원칙은 스테이커 보호와 L2 성장 인센티브의 균형을 달성하며, 쌍곡선 함수는 시장 충격을 최소화한다. 토카막 네트워크 사례 연구는 프레임워크가 실제 네트워크에 적용 가능함을 입증하며, L2 전환을 계획하는 다른 시뇨리지 발행 블록체인에도 적용될 수 있음을 시사한다.

## 부록 A: 수학적 증명

### A.1 정리: 시장 충격의 단조 감소

B > B_r에서 스테이커 APY의 B에 대한 민감도를 계산한다.

$$\frac{d(APY_S)}{dB} = \frac{A_{total} \cdot \sigma}{S} \cdot \frac{d\lambda}{dB}$$

$$= \frac{A_{total} \cdot \sigma}{S} \cdot \left(-(1-\lambda_{min}) \cdot \frac{k}{(k+B-B_r)^2}\right)$$

충격의 크기 $|\frac{d(APY_S)}{dB}|$는 $(k+B-B_r)^2$에 반비례한다. B가 증가함에 따라 분모가 증가하므로 충격의 크기는 단조 감소한다.

### A.2 λ_min 도출

목표 최소 APY에서 λ_min을 역산하는 과정은 다음과 같다.

최종 상태(B → ∞)에서 r = 0이므로 스테이커 APY는 지분 시뇨리지만으로 결정된다.

$$APY_{min} = \frac{A_{total} \cdot \lambda_{min} \cdot \sigma}{S}$$

이를 λ_min에 대해 정리하면:

$$\lambda_{min} = \frac{APY_{min} \cdot S}{A_{total} \cdot \sigma}$$

토카막 네트워크 파라미터를 대입하면(APY_min = 5%):

$$\lambda_{min} = \frac{0.05 \times 27,234,159}{10,308,816 \times 0.274} \approx 0.48$$

실용적 편의를 위해 0.5로 설정한다.

## 부록 B: 시뮬레이션 코드

```python
import numpy as np
import matplotlib.pyplot as plt

class DualIncentiveFramework:
    """동적 이중 인센티브 최적화 프레임워크"""

    def __init__(self, A_total, S, sigma, r_0, B_r, lambda_min, k):
        self.A_total = A_total      # 연간 시뇨리지
        self.S = S                  # 총 스테이킹
        self.sigma = sigma          # 스테이킹 비율
        self.r_0 = r_0              # 초기 추가 시뇨리지율
        self.B_r = B_r              # r 제거 임계점
        self.lambda_min = lambda_min # 최소 지분 팩터
        self.k = k                  # 반포화 상수

    def r(self, B):
        """추가 시뇨리지율: r 선감소"""
        return self.r_0 * max(0, 1 - B / self.B_r)

    def lambda_(self, B):
        """지분 시뇨리지율: r=0 이후 감소 시작"""
        excess = max(0, B - self.B_r)
        return self.lambda_min + (1 - self.lambda_min) * self.k / (self.k + excess)

    def APY_staker(self, B):
        """스테이커 APY"""
        lam = self.lambda_(B)
        r = self.r(B)
        staked_seig = lam * self.sigma
        relative_seig = r * (1 - lam * self.sigma)
        return (self.A_total * (staked_seig + relative_seig)) / self.S

    def A_L2(self, B, d=0.1):
        """L2 분배 재원"""
        lam = self.lambda_(B)
        r = self.r(B)
        return self.A_total * (1 - lam * self.sigma) * (1 - r) * (1 - d)

# 사용 예시 (토카막 네트워크 파라미터)
framework = DualIncentiveFramework(
    A_total=10_308_816,
    S=27_234_159,
    sigma=0.274,
    r_0=0.4,
    B_r=20_000_000,
    lambda_min=0.5,
    k=10_000_000
)

# 시뮬레이션
B_values = [0, 10e6, 20e6, 30e6, 50e6, 100e6]
print("B(M)\tr(B)\tλ(B)\tAPY_S\tA_L2(M)")
for B in B_values:
    print(f"{B/1e6:.0f}\t{framework.r(B):.2f}\t{framework.lambda_(B):.2f}\t"
          f"{framework.APY_staker(B)*100:.1f}%\t{framework.A_L2(B)/1e6:.2f}")
```
